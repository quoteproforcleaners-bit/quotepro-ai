HIGH-LEVEL ARCHITECTURE
==================================================

Use this architecture:

Customer texts business number
-> Linq receives inbound message
-> Linq webhook calls my Express backend
-> backend loads business/customer/quote context
-> backend classifies message intent
-> backend decides:
   - continue intake
   - answer FAQ
   - generate quote draft
   - human handoff
-> backend stores transcript/state in Postgres
-> backend replies via Linq API
-> mobile/web app show shared inbox + automation status

- ask one question at a time during intake
- do not invent pricing
- do not promise availability unless app has that info

- user asks for a human

- discount request
- commercial cleaning request
- AI confidence below threshold
- unsupported question
- payment dispute
- reschedule/cancellation edge case if business rules unclear

- mark handoff_status = human
- pause AI replies
