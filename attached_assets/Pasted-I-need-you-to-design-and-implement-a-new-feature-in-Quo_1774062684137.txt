I need you to design and implement a new feature in QuotePro:

Customer Self-Booking Portal
Clients should be able to book cleaning slots themselves 24/7 from the branded link.

Goal:
Create a premium, native self-booking experience inside QuotePro. Do NOT use Calendly unless absolutely necessary as a temporary fallback. This should be built as a QuotePro-native feature tied to our quotes, customers, and calendar.

Main use case:
When a customer accepts a quote, they should be able to immediately self-book their cleaning by choosing from available dates/time windows.

Secondary use case:
This feature should be architected so it can later support open booking directly from the branded link for new customers too.

What I need:
1. Audit the current branded link flow, quote flow, accepted-quote flow, and calendar/scheduling logic
2. Design the best architecture for a self-booking portal
3. Implement a customer-facing booking page on the branded link
4. Let customers choose from real available cleaning slots
5. Save the booking to QuotePro
6. Add the booked clean to the calendar automatically
7. Tie the booking to the correct quote/customer/job record
8. Send confirmation after booking
9. Make the UX feel premium, modern, and very simple

Important:
Do NOT treat this like a generic appointment scheduler.
This is for cleaning businesses, so the system needs cleaning-specific availability rules.

Availability should support rules like:
- allowed business days
- allowed time windows
- blackout dates
- minimum notice required
- max jobs per day
- service area restrictions
- optional future crew/capacity logic

MVP requirements:
- Start with accepted-quote self-booking
- Customer accepts quote
- Customer is prompted to book their clean
- They see available dates/time windows
- They choose a slot
- Booking is confirmed
- The clean appears on the internal QuotePro calendar
- Customer, price, address, service type, and quote/job info are all preserved

Portal requirements:
- branded and customer-facing
- clean and modern
- mobile-friendly
- premium look and feel
- easy to use
- clear confirmation state
- no clutter

Data requirements:
- booking must be tied to accepted quote
- avoid duplicate bookings for the same accepted quote
- rescheduling should be possible later
- booking should update quote/job status appropriately

Please also decide and explain:
- whether Calendly is needed or not
- whether this should be fully native
- how availability should be stored
- how slot generation should work
- how to keep the architecture scalable for future open booking from the branded link

Deliverables:
1. Explain the recommended architecture in plain English
2. Identify the exact files/functions to update
3. Implement the MVP
4. Show how the customer-facing self-booking flow works
5. Show how bookings land on the QuotePro calendar
6. Explain how to test it end to end
7. List future enhancements for phase 2 and phase 3

Success criteria:
- Customers can self-book 24/7 from the branded link after accepting a quote
- No Calendly dependency unless absolutely necessary
- Booking feels native and premium
- Internal team phone time is reduced
- Scheduled cleans automatically populate in QuotePro calendar
- Architecture is clean and ready for future expansion
