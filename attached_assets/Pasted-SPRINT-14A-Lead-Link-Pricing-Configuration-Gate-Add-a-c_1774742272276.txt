SPRINT 14A — Lead Link Pricing Configuration Gate

Add a configuration detection system that warns 
software users when their pricing isn't set up, 
and shows a disclaimer on the Lead Link microsite 
when default pricing is being used.

DEFINITION OF "PRICING CONFIGURED":
A business owner is considered "pricing configured" 
when ALL of the following are true:
  1. hourlyRate is set AND > 0 AND != default value (35)
  2. minimumTicket is set AND > 0 AND != default value (100)
  3. At least one serviceType has been customized
     (name is not one of the factory defaults)
  4. At least 3 add-on prices have been set to non-zero

Create a server-side function isPricingConfigured(userId) 
that checks these conditions and returns:
{
  configured: boolean,
  missingItems: string[], // list of what's not set up
  completionPercent: number // 0-100
}

Store the result in the lead_link_config API response 
so the microsite can access it.
