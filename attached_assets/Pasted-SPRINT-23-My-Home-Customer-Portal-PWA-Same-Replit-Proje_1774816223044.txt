SPRINT 23 — "My Home" Customer Portal
(PWA — Same Replit Project, New Public Routes)

Build a branded, mobile-first customer portal 
that gives homeowners a beautiful, permanent 
window into their cleaning service. This is 
a set of new public-facing routes added to 
the existing Express/React project.

NO new Replit project. NO separate database.
Uses existing jobs, customers, photos, 
team_members, tips, and quotes tables.

═══════════════════════════════
ARCHITECTURE OVERVIEW
═══════════════════════════════

NEW DATABASE TABLE: customer_portals
  id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  customer_id INTEGER REFERENCES customers(id)
  user_id INTEGER REFERENCES users(id)
  token VARCHAR(64) UNIQUE NOT NULL
  created_at TIMESTAMPTZ DEFAULT NOW()
  last_viewed_at TIMESTAMPTZ
  view_count INTEGER DEFAULT 0

Token generation:
  Use crypto.randomBytes(32).toString('hex')
  This gives a 64-character hex string
  Permanent — never expires
  One token per customer per business

NEW API ROUTES (public — no auth middleware):
  GET /api/portal/:token
    Returns all portal data for this customer
    
  PUT /api/portal/:token/preferences
    Customer updates their home preferences
    
  POST /api/portal/:token/reschedule
    Customer requests a reschedule
    
  GET /api/portal/:token/photos/:jobId
    Returns photos for a specific job

NEW FRONTEND ROUTES (public pages):
  /home/:token → CustomerPortalPage
  /home/:token/photos → PhotoGalleryPage
  /home/:token/preferences → PreferencesPage
  /home/:token/tip → TipPage (reuse Sprint 21B)
  /home/:token/reschedule → ReschedulePage

These routes must be excluded from the 
main app's auth guard. Add them to the 
public routes list alongside /trial, 
/pricing, /j/:token (live update page).

═══════════════════════════════
TOKEN GENERATION — WHEN & HOW
═══════════════════════════════

Generate a portal token automatically when:
  1. A customer is created (new)
  2. The first job is scheduled for an 
     existing customer with no token
  3. Business owner manually clicks 
     "Send Portal Link" from customer profile

On generation:
  INSERT into customer_portals 
    (customer_id, user_id, token)
  If token already exists for this 
  customer_id + user_id pair: return 
  existing token (idempotent)

BACKFILL: When this sprint deploys, 
run a one-time migration to generate 
tokens for all existing customers 
who don't have one yet.

═══════════════════════════════
API ENDPOINT: GET /api/portal/:token
═══════════════════════════════

This is the main data endpoint.
No authentication header required —
token IS the authentication.

VALIDATE token:
  SELECT cp.*, c.*, u.*
  FROM customer_portals cp
  JOIN customers c ON cp.customer_id = c.id
  JOIN users u ON cp.user_id = u.id
  WHERE cp.token = :token
  
  If not found: return 404
  If found: update last_viewed_at, 
  increment view_count

RETURN this shape:
{
  customer: {
    firstName: string,
    lastName: string,
    address: string,
    phone: string,
    preferences: object | null,
  },
  
  business: {
    name: string,
    phone: string,
    logoUrl: string | null,
    primaryColor: string | null,
    senderName: string,
  },
  
  nextJob: {
    id: string,
    startDatetime: string,
    jobType: string,
    assignedCleaner: {
      firstName: string,
      photoUrl: string | null,
      bio: string | null,
    } | null,
    status: string,
    detailedStatus: string,
  } | null,
  
  lastJob: {
    id: string,
    completedAt: string,
    jobType: string,
    total: number | null,
    assignedCleaner: {
      firstName: string,
      photoUrl: string | null,
    } | null,
    photos: {
      id: string,
      photoUrl: string,
      photoType: "before" | "after",
      caption: string | null,
    }[],
    satisfactionRating: number | null,
    tipAmount: number | null,
  } | null,
  
  upcomingJobs: {
    id: string,
    startDatetime: string,
    jobType: string,
    assignedCleaner: {
      firstName: string,
    } | null,
  }[],  // Next 3 jobs max
  
  jobHistory: {
    id: string,
    startDatetime: string,
    completedAt: string | null,
    jobType: string,
    total: number | null,
    photoCount: number,
    satisfactionRating: number | null,
    tipAmount: number | null,
  }[],  // Last 10 jobs
  
  outstandingBalance: {
    amount: number,
    invoiceId: string,
    dueDate: string,
  } | null,
  
  recurrence: string | null,
  // "biweekly", "monthly" etc — shown 
  // as "Your home is cleaned every 2 weeks"
  
  portalToken: string,
  // Included so client can construct URLs
}

SECURITY NOTES:
- Never return internal IDs in responses
  Use the portal token for all actions
- Never return other customers' data
- Rate limit: 60 requests per hour per IP
- Do not return team member phone/email
- Do not return business owner email

══════════════════════════════
PWA — "ADD TO HOME SCREEN"
══════════════════════════════

Add PWA support so customers can 
install the portal on their phone:

1. CREATE /public/manifest.json:
{
  "name": "[BusinessName] Portal",
  "short_name": "My Cleaning",
  "start_url": "/home/[token]",
  "display": "standalone",
  "background_color": "#ffffff",
  "theme_color": "[businessPrimaryColor]",
  "icons": [
    {
      "src": "[businessLogoUrl or default]",
      "sizes": "192x192",
      "type": "image/png"
    }
  ]
}

Note: manifest is generated dynamically 
per portal (different business name/color)
Route: GET /portal-manifest/:token
Returns the manifest JSON with business 
details filled in.

2. Add to portal HTML head:
  <link rel="manifest" 
    href="/portal-manifest/[token]">
  <meta name="apple-mobile-web-app-capable" 
    content="yes">
  <meta name="apple-mobile-web-app-title" 
    content="My Cleaning">
  <meta name="theme-color" 
    content="[businessPrimaryColor]">

3. SHOW "Add to Home Screen" PROMPT:
  After the customer's second visit 
  (view_count >= 2):
  
  Show a subtle bottom banner:
  Background: white
  Border-top: 1px solid #e2e8f0
  Padding: 12px 16px
  
  Left: home screen icon (📱)
  Center: 
    "Add to your home screen"
    "Get quick access to your 
    portal anytime"
  Right: [Add] button → triggers 
    browser install prompt
  
  Dismiss X — stores in localStorage:
  key: "portal_install_dismissed"
  Never shows again after dismiss.

  It creates a reschedule_requests

  Push notification:
  "[CustomerFirstName] wants to 
  reschedule their [Date] clean"
  
  Email to business owner with 
  the requested new date and note.
  
  Confirmation to customer:
  "Your reschedule request has been sent! 
  [BusinessName] will confirm shortly."

CREATE reschedule_requests table:
  id, portal_token, job_id, 
  requested_date, preferred_time,
  customer_note, status (pending|confirmed|declined),
  created_at

PORTAL LINK DELIVERY
═══════════════════════════════

The portal link should be sent 
automatically in these moments:

1. AFTER JOB COMPLETED:
   The existing post-job SMS (Sprint 20A) 
   should include the portal link:
   
   "Hi [FirstName]! Maria just finished 
   your home ✨ See your clean summary 
   and photos: [portal-link]"

2. WHEN QUOTE IS ACCEPTED:
   SMS to customer:
   "Hi [FirstName]! Your cleaning is 
   confirmed. Track everything here: 
   [portal-link]
   See you [date]!"

3. APPOINTMENT REMINDER (Sprint 20A):
   Include portal link:
   "Your cleaning is tomorrow at [time]. 
   View details: [portal-link]"

4. MANUAL SEND from customer profile:
   Button in the business owner's 
   customer detail page:
   "📱 Send Portal Link"
   → Sends SMS with portal link
   → Shows confirmation toast

PORTAL LINK FORMAT:
  https://getquotepro.ai/home/[token]
  
  NOT a Replit subdomain.
  This must be on the production domain.
