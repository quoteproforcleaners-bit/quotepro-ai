═══════════════════════════════
HOW THE TIP SYSTEM WORKS
═══════════════════════════════

FLOW:
1. Job is marked as completed
2. System automatically sends customer 
   a "How was your clean?" SMS/email
   (this is the tip request — subtle, 
   not pushy)
3. Customer clicks link → lands on 
   tip page
4. Customer selects tip amount or 
   enters custom amount
5. Customer pays via Stripe
6. Business owner and cleaner are 
   notified
7. Tip is recorded in the system

═══════════════════════════════
DATABASE CHANGES
═══════════════════════════════

1. ADD to user settings:
   tipsEnabled: boolean (default: false)
   tipPercentageOptions: integer[] 
     (default: [18, 22, 25])
   tipDistributionPercent: integer 
     (default: 100 — goes to cleaner)
     Range: 0-100
   tipRequestDelay: integer 
     (hours after job completion, default: 2)

2. CREATE tips table:
   id UUID PRIMARY KEY
   job_id INTEGER REFERENCES jobs(id)
   customer_id INTEGER REFERENCES customers(id)
   team_member_id INTEGER (who receives it)
   amount DECIMAL(10,2)
   percentage DECIMAL(5,2)
   stripe_payment_intent_id TEXT
   status VARCHAR(20) 
     (pending|completed|failed|refunded)
   tip_page_viewed_at TIMESTAMPTZ
   created_at TIMESTAMPTZ
   paid_at TIMESTAMPTZ

3. ADD to jobs table:
   tip_request_sent_at TIMESTAMPTZ
   tip_amount DECIMAL(10,2)

═══════════════════════════════
STRIPE INTEGRATION NOTES
═══════════════════════════════

Tips require Stripe to be connected 
to the business owner's account.

If Stripe is not connected:
  - Disable tips toggle
  - Show message: "Connect Stripe 
    to start collecting tips"
  - Link to Stripe connection flow

Payment flow:
  - Create PaymentIntent for tip amount
  - Use business owner's Stripe account 
    (not QuotePro's) so money goes 
    directly to them
  - QuotePro does NOT take a cut of tips
    (important trust point — mention in 
    the onboarding step description)
  - Webhook: payment_intent.succeeded → 
    update tip status to "completed"
