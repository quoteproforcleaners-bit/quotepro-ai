   In server/middleware.ts, in the requireAuth middleware,
   after successful auth:
   
   UPDATE users SET last_active_at = NOW() 
   WHERE id = req.user.id
   
   Do this asynchronously (don't await) so it never 
   adds latency to the request.

3. CHURN RISK SCORING CRON
   Add to server/index.ts, runs daily at 6am:
   
   For every paid user, compute churn_risk_score (0-100):
   
   Start at 0, add points for risk signals:
   +30: No login in 7 days (paid user)
   +20: No quote sent in 14 days
   +15: Hit quote quota and didn't upgrade within 48h
   +15: Opened cancel flow (CANCEL_INITIATED event exists)
   +10: No login in 3 days (during trial)
   +10: Trial has 2 days left, not upgraded
   +5: Zero jobs scheduled (no quote ever accepted)
   
   Subtract points for health signals:
   -20: Quote accepted in last 7 days
   -15: Job completed in last 7 days  
   -10: AI agent used in last 3 days
   -10: Logged in today
   
   Cap at 0 minimum, 100 maximum.
   Update churn_risk_score in users table.
