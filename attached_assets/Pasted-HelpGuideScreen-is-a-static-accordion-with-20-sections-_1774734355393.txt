HelpGuideScreen is a static accordion with 20+ sections 
and no way to search. Add a search input that filters 
sections by keyword.

REQUIREMENTS:

1. ADD SEARCH INPUT at the top of the screen, 
   below the header card:

<View style searchContainer>
  <Feather name="search" size={16} color={theme.textMuted} />
  <TextInput
    placeholder="Search the guide..."
    value={searchQuery}
    onChangeText={setSearchQuery}
    style={searchInput}
    clearButtonMode="while-editing"  // iOS only
    autoCorrect={false}
  />
  {searchQuery.length > 0 on Android: show X button}
</View>

Style: 
  background: theme.backgroundSecondary
  border: 1px solid theme.border
  borderRadius: BorderRadius.full (pill shape)
  padding: 10px 16px
  marginBottom: Spacing.lg

2. FILTER LOGIC:
Add state: const [searchQuery, setSearchQuery] = useState("")
Add debounce: useDebounce(searchQuery, 200)

Filter sections:
const filteredSections = useMemo(() => {
  if (!debouncedSearch.trim()) return sections;
  const q = debouncedSearch.toLowerCase();
  return sections.filter(section =>
    section.title.toLowerCase().includes(q) ||
    section.content.some(item => 
      item.toLowerCase().includes(q)
    )
  );
}, [sections, debouncedSearch]);

Replace sections.map with filteredSections.map

3. HIGHLIGHT MATCHES:
When searching, auto-expand sections that contain matches:
- If filteredSections.length <= 3, auto-expand all of them
- Otherwise keep existing expand/collapse behavior

4. EMPTY STATE when no results:
<View style centered>
  <Feather name="search" size={24} color={theme.textMuted} />
  <ThemedText>No results for "[searchQuery]"</ThemedText>
  <ThemedText type="caption">
    Try a different keyword or browse all sections below
  </ThemedText>
  <Pressable onPress={() => setSearchQuery("")}>
    Clear search
  </Pressable>
</View>

5. RESULT COUNT when searching:
Show below search input:
"[n] section[s] match '[query]'"
Only visible when searchQuery is active.

6. CLEAR BEHAVIOR:
- Clearing the search restores all sections
- Collapse all sections when search is cleared
  (reset expandedId to null)

Test: Searching "stripe" shows the Integrations section.
Searching "follow up" shows Follow-Up Queue section.
No matches shows the empty state.
