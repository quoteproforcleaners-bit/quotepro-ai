2. NEW API ROUTES (add to server/routers/integrationsRouter.ts)

   POST /api/subscriptions/create-checkout-session
   - requireAuth middleware
   - Body: { priceId, successUrl, cancelUrl }

   - Set customer_email from the authenticated user's email
   - Set metadata: { userId: req.user.id }
   - Set allow_promotion_codes: true
   - Return { url } — the Stripe Checkout URL

   POST /api/subscriptions/create-portal-session
   - requireAuth middleware  
   - Creates a Stripe Billing Portal session for the user
   - Return { url } — the portal URL for managing/canceling

   POST /api/subscriptions/webhook
   - Raw body parser (NOT json parser)
   - Verify HMAC with STRIPE_WEBHOOK_SECRET
   - Handle these events:
     checkout.session.completed → update users.subscriptionTier
     customer.subscription.updated → sync tier changes
     customer.subscription.deleted → downgrade to 'free'
     invoice.payment_failed → send email via SendGrid warning user
   - Map Stripe Price IDs to tier names:
     starter_monthly → 'starter'
     growth_monthly / growth_annual → 'growth'  
     pro_monthly / pro_annual → 'pro'
   - Store stripe_customer_id on the users table
