I have a pricing engine for a cleaning business SaaS app.
After Sprint 3, the canonical engine lives in shared/pricingEngine.ts.

Set up a test suite and write comprehensive tests:

1. TEST SETUP
   - Install vitest and @vitest/coverage-v8
   - Add to package.json: "test": "vitest", "coverage": "vitest --coverage"
   - Create vitest.config.ts

2. WRITE TESTS in shared/pricingEngine.test.ts
   Cover ALL of these cases:

   BASE PRICE CALCULATION
   - 1000 sqft, 2 bed, 1 bath, no pets, average condition → expected base
   - 2500 sqft, 4 bed, 2.5 bath → expected base
   - Minimum ticket enforcement (tiny property should never go below minimum)

   CONDITION MULTIPLIERS
   - Score 9+ → 0.9x multiplier applied correctly
   - Score 7-8 → 1.0x (no change)
   - Score 5-6 → 1.2x applied correctly
   - Score 3-4 → 1.4x applied correctly
   - Score <3 → 1.7x applied correctly

   SERVICE TIER MULTIPLIERS
   - Good tier (0.75x) produces correct price
   - Better tier (1.0x) produces correct price
   - Best tier (1.5x) produces correct price
   - $20 minimum gap is enforced between all three tiers
   - Tiers are always in ascending price order

   FREQUENCY DISCOUNTS
   - Weekly: 25% discount applied to recurring price
   - Bi-weekly: 15% discount applied
   - Monthly: 10% discount applied
   - One-time: no discount applied
   - First clean price is always higher than recurring price

   ADD-ONS
   - Each add-on adds its flat price to the total
   - Multiple add-ons stack correctly
   - Tier-included add-ons do not add to the price

   ROUNDING
   - All prices round to the nearest $5
   - $142 → $140, $143 → $145

   PETS
   - Pet surcharge is added correctly
   - Multiple pets stack correctly

   EDGE CASES
   - Zero sqft input → returns minimum ticket
   - Negative values → throws or returns minimum ticket
   - All add-ons selected simultaneously → correct total
   - Extreme condition (score 1) + large property + all add-ons 
     → correct total without overflow

3. COVERAGE REQUIREMENT
   Configure coverage to fail if line coverage drops below 90% 
   on shared/pricingEngine.ts
