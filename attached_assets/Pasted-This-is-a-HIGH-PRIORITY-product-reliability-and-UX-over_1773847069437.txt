This is a HIGH-PRIORITY product, reliability, and UX overhaul for the QuotePro “Quote from Notes” feature.

Right now the feature is broken and underpowered.
It currently throws an “Internal Server Error,” which destroys trust.
This feature should become one of the most valuable workflows in QuotePro:
paste messy notes, walkthrough text, customer messages, or property descriptions, and AI turns them into a usable quote draft fast.

Do NOT treat this as a bug-only fix.
We need:
1. root-cause debugging for the server error
2. a much better extraction experience
3. a more premium, reliable AI workflow
4. structured output that is actually useful for quoting

Do not break quotes, CRM, pricing logic, subscriptions, or other AI tools.
Before coding, first inspect the current implementation and identify:
- where the “Quote from Notes” screen/component lives
- what backend route / server action / API it calls
- what prompt is sent to AI
- why the current request is returning Internal Server Error
- whether the error is frontend-only, backend-only, validation-related, or model-related
- whether there are missing env vars / API keys / malformed JSON / timeout issues
- whether parsing of AI output is failing
- whether the app expects a structure the model is not returning
- whether quote creation is happening automatically after extraction or only displaying parsed details
Then implement carefully.

==================================================
CORE PRODUCT DIRECTION
==================================================

This feature should help a cleaning business owner do this:

“I have messy notes from a walkthrough, text thread, phone call, or property description. Turn this into a usable quote draft.”

The AI should extract key details like:
- bedrooms
- bathrooms
- square footage if mentioned
- cleaning type
- deep clean vs standard vs move in/out
- frequency if mentioned
- pets
- occupants
- add-ons / special requests
- condition / dirtiness
- location / address if mentioned
- commercial vs residential clues
- missing info / ambiguities

Then it should turn that into:
- structured extracted details
- recommended quote assumptions
- a quote draft or prefilled quote form
- clear next steps if information is missing

This should feel like a real quoting productivity tool, not a raw text parser.

==================================================
PRIMARY GOALS
==================================================

1. Fix the Internal Server Error completely
2. Make note extraction accurate and reliable
3. Make the output structured and useful
4. Help the user go from messy note → quote draft fast
5. Make the workflow feel premium and trustworthy

==================================================
PART 1 — FIX THE INTERNAL SERVER ERROR
==================================================

First, identify the real root cause of the current failure.

Possible causes to inspect:
- bad request body shape
- missing required fields
- malformed AI response
- JSON parse failure
- route exception
- timeout
- missing API key
- model failure
- trying to create quote with invalid / null values
- mismatch between extracted fields and frontend expectations
- zod/schema validation failure
- prompt returning prose instead of structured output

Do not just replace the error message.
Fix the real issue.

Then improve error handling:
- log useful debugging info
- return actionable error states
- show helpful user-facing messages
- never show a generic raw “Internal Server Error” unless absolutely unavoidable

==================================================
PART 2 — REDESIGN THE WORKFLOW
==================================================

The feature should be organized around a clearer workflow:

Step 1:
User pastes notes / walkthrough text / customer message / property description

Step 2:
AI extracts structured details

Step 3:
User reviews the extracted details

Step 4:
User confirms / edits assumptions

Step 5:
AI or app turns the extracted info into a quote draft

The current empty right panel is too weak.
It should become a structured “AI Extraction” results panel.

==================================================
PART 3 — STRUCTURED EXTRACTION OUTPUT
==================================================

Do NOT output a blob of text.

The extracted result should be displayed in clear sections/cards, for example:

1. Property Type
- Residential / Commercial
- confidence if useful

2. Property Details
- bedrooms
- bathrooms
- square footage
- occupants
- pets

3. Service Details
- standard clean / deep clean / move in-out / recurring / one-time
- frequency
- condition level
- urgency / timing if mentioned

4. Special Requests / Add-ons
- fridge
- oven
- laundry
- windows
- pet hair
- organization
- etc.

5. Missing Information
- what the AI could not determine
- what the user should confirm before finalizing quote

6. Recommended Quote Setup
- suggested cleaning type
- suggested frequency assumption if any
- suggested upsell or caution if appropriate

This should render in a polished structured UI, not as raw text.

==================================================
PART 4 — BETTER NOTE INPUT EXPERIENCE
==================================================

Improve the left-side input area.

The user should know what kinds of input this supports:
- walkthrough notes
- text threads
- property descriptions
- voicemail transcription
- rough lead notes
- realtor / prospect requests

Improve placeholder examples, such as:
- “3 bed, 2 bath home in Malvern. First-time clean, 1 dog, customer wants biweekly if price makes sense.”
- “Move-out clean for 2,100 sq ft condo. Needs inside oven and fridge. Very dirty kitchen.”
- “Customer texted: standard clean every 2 weeks, 4 bed, 3 bath, 2 cats, wants quote by Friday.”

Optional but highly valuable:
- add example chips the user can tap to auto-fill sample note formats
- allow “Clear” action
- maybe support paste from clipboard elegantly

==================================================
PART 5 — MAKE EXTRACTION SMARTER FOR CLEANING BUSINESSES
==================================================

The AI must understand cleaning-specific domain logic.

It should recognize and infer from phrases like:
- first-time clean
- maintenance clean
- deep clean
- recurring
- standard clean
- move-in / move-out
- pet hair
- very dirty / hasn’t been cleaned in months
- wants every two weeks
- only bathrooms and floors
- needs fridge and oven
- commercial office / boutique / salon / Airbnb / rental turnover

It should not answer like a generic data extraction bot.
It should behave like a smart quoting assistant for cleaning businesses.

==================================================
PART 6 — HANDLE MISSING OR PARTIAL INFORMATION WELL
==================================================

Often notes will be incomplete.
Do NOT fail just because info is missing.

Instead:
- extract what is known
- mark missing fields clearly
- suggest assumptions carefully
- let user continue editing

Examples:
If square footage missing:
- do not crash
- flag as missing
- allow user to fill in later

If bedrooms/bathrooms present but no service type:
- infer likely intent if possible
- mark confidence / uncertainty

If note suggests deep clean:
- say why
- e.g. “first-time clean + high dirtiness language suggests deep clean”

==================================================
PART 7 — CREATE A REAL QUOTE DRAFT HANDOFF
==================================================

This feature should not stop at extraction.

After extraction, there should be a clear next action:
- “Create Quote Draft”
- “Use Extracted Details”
- “Send to Quote Builder”

The app should prefill the quote workflow with extracted data where possible.

If the current architecture already supports populating quote fields, wire it up properly.
If not, create the cleanest handoff possible.

Do not force the user to manually retype everything after extraction.

==================================================
PART 8 — BETTER EMPTY / LOADING / ERROR STATES
==================================================

Current right-side panel is weak.

Improve states:

Empty state:
- explain what the feature does
- e.g. “Paste notes on the left and AI will extract quote details for you.”

Loading state:
- “Reading notes...”
- “Extracting property details...”
- “Building quote assumptions...”
Use polished loading skeletons or staged progress if helpful

Error state:
- explain what went wrong in plain English
- allow retry
- preserve user input
- do not wipe note text on failure
- do not show vague raw backend messaging unless logged for debugging only

==================================================
PART 9 — OUTPUT SHOULD FEEL PREMIUM
==================================================

This should not feel like a cheap parser tool.

The result should feel:
- trustworthy
- clear
- fast
- practical
- premium
- built for operators who want to move quickly

Use strong visual hierarchy, clear cards/rows, and clean labels.

==================================================
PART 10 — GATING / PLAN LOGIC
==================================================

Audit whether this feature should be gated.

My recommendation:
This should be at least a Growth feature.
If that aligns with current product strategy, enforce it clearly.

If user is not entitled:

- explain that AI note extraction is included in Growth
- provide upgrade CTA

Do not leave access ambiguous.
