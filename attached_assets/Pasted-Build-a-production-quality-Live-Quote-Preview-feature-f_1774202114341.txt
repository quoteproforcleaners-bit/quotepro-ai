Build a production-quality “Live Quote Preview” feature for the QuotePro web app.

This feature should show the quote amount updating in real time while the software user is building a quote.

Important:
- This is for the web app
- It must support both residential and commercial quotes
- The displayed amount must increase/decrease instantly as inputs change
- The live amount must use the same pricing engine as the final saved quote
- This should feel premium, fast, and trustworthy

PRODUCT GOAL

As the user creates a quote, they should always be able to see a live pricing panel that updates in real time based on the quote inputs.

Examples:
- changing square footage updates the amount immediately
- adding bathrooms increases the amount
- changing frequency changes the amount
- adding a deep clean multiplier changes the amount
- adding commercial restrooms, floor care, or after-hours requirements changes the amount
- manual adjustments change the total immediately

This should feel like a serious quoting engine, not a simple calculator.

CORE FEATURE NAME

Live Quote Preview

HIGH-LEVEL UX

Build the quote creation experience so that:

1. The quote builder form is on the left/main area
2. A sticky “Live Quote Preview” panel appears on the right on desktop
3. As fields are changed, the quote amount updates immediately
4. The preview panel shows:
   - current total
   - breakdown of price components
   - triggered pricing rules
   - discounts/surcharges
   - estimated hours if available
   - warnings if pricing is below minimum or outside expected range
5. The amount shown in the preview should match the final quote amount when the quote is saved

For smaller screens, translate this into a mobile-friendly layout if needed, but prioritize web app desktop UX first.

REQUIRED SUPPORT

The feature must support both:
- residential quoting
- commercial quoting

These should share a common pricing engine architecture but support different rule/input models.

PRICING ENGINE REQUIREMENTS

This is critical:
The live preview must use the same deterministic pricing engine as final quote creation.

Do NOT build:
- one set of logic for the UI preview
- and another set of logic for final quote save

There should be one shared source of truth.

Implement a pricing engine that:
- accepts quote input data
- evaluates pricing rules
- calculates total
- calculates line-item breakdown
- returns triggered rules
- supports residential and commercial pricing logic
- supports draft vs published pricing profiles if relevant
- supports manual adjustments
- is deterministic and testable

FEATURE BEHAVIOR

As the user changes quote inputs, the UI should update:
- instantly or near-instantly
- without requiring page refresh
- without requiring user to click “calculate”

Use debounced/reasonable recalculation if needed, but the UX should feel immediate.

LIVE QUOTE PREVIEW PANEL REQUIREMENTS

The preview panel should include:

1. Current Total
- large, prominent dollar amount
- animated subtly when changed
- visually obvious and trustworthy

2. Price Breakdown
Show major components such as:
- base price
- square footage contribution
- room/bath adjustments
- service-type multiplier
- frequency discount
- condition surcharge
- pet surcharge
- add-ons
- travel surcharge
- custom adjustments
- commercial-specific charges

3. Triggered Pricing Rules
Show which rules affected the quote, for example:
- minimum job price applied
- deep clean multiplier applied
- monthly discount applied
- restroom count adjustment applied
- after-hours surcharge applied

4. Hours / Effort Estimate
If the pricing engine supports it, show:
- estimated labor hours
- crew assumptions
- per-visit estimate

5. Warnings / Guardrails
Examples:
- below minimum recommended price
- quote outside normal range
- missing required inputs
- very low commercial estimate
- unusually high discount applied

6. Manual Adjustments
Allow the user to add:
- fixed-dollar adjustment
- percentage adjustment if supported
- reason/note
These should instantly update the live total.
