We need to improve the customer-facing job status / update page flow inside QuotePro.

CURRENT PROBLEM
Right now the cleaner has to manually tap every stage:
- En Route
- Service Started
- In Progress
- Final Touches
- Completed

This is too much manual work and creates friction.

NEW GOAL
Change the flow so only these stages are manual:
1. En Route
2. Service Started
3. Completed

The middle progress stages should update automatically based on time elapsed after "Service Started".

NEW FLOW

Manual steps:
- Cleaner taps "En Route"
- Cleaner taps "Service Started"
- Cleaner taps "Completed" when the job is finished

Automatic steps:
After "Service Started", the system should automatically move the customer-facing progress page through intermediate stages based on the job type and elapsed time.

SERVICE TYPES AND TIMING RULES

1. Regular Clean
- Auto-update every 30 minutes

2. Deep Clean
- Auto-update every 45 minutes

3. Move In / Move Out
- Auto-update every 45 minutes

PROGRESS STAGES

Suggested customer-facing progress flow:
- Scheduled
- En Route
- Service Started
- In Progress
- Final Touches
- Completed

IMPORTANT LOGIC
- "Scheduled" remains the default state before anything starts
- "En Route" only changes when cleaner manually taps it
- "Service Started" only changes when cleaner manually taps it
- After "Service Started", the app should automatically update:
  - first auto state: In Progress
  - second auto state: Final Touches
- "Completed" only changes when cleaner manually taps it

AUTO-PROGRESSION RULES

For Regular Clean:
- At service start: show Service Started
- After 30 minutes: auto-update to In Progress
- After 60 minutes: auto-update to Final Touches
- Completed only when cleaner manually taps Completed

For Deep Clean:
- At service start: show Service Started
- After 45 minutes: auto-update to In Progress
- After 90 minutes: auto-update to Final Touches
- Completed only when cleaner manually taps Completed

For Move In / Move Out:
- At service start: show Service Started
- After 45 minutes: auto-update to In Progress
- After 90 minutes: auto-update to Final Touches
- Completed only when cleaner manually taps Completed

TECHNICAL REQUIREMENTS

1. Add a job/service type field if needed:
- regular_clean
- deep_clean
- move_in_out

2. Store timestamps for:
- en_route_at
- service_started_at
- completed_at
- current_status_last_updated_at

3. Add logic that derives or updates the current customer-facing stage based on:
- service type
- service_started_at
- current time
- completed_at

4. Use a background job / scheduler / polling-safe approach if the app already supports this
5. If background jobs are not available, compute the current stage dynamically on page load and refresh so the customer still sees the right status
6. Do not require constant manual taps from the cleaner

UI REQUIREMENTS

Cleaner-facing UI:
- only show buttons for:
  - En Route
  - Service Started
  - Completed
- do not show manual buttons for In Progress or Final Touches

Customer-facing update page:
- continue to show the full progress timeline
- timeline should automatically advance based on logic above
- keep the page visually polished and reassuring
- preserve existing design system

TIMELINE / STATUS HISTORY
- Keep timeline history entries
- Add automatic timeline events when the system changes:
  - In Progress
  - Final Touches
- Mark those entries as system-generated if helpful internally
- Display them normally to the customer

EDGE CASES
- If cleaner taps Completed early, job immediately becomes Completed
- If service type is missing, default to regular_clean timing
- If Service Started has not been tapped yet, do not auto-progress
- If Completed is tapped, no further automatic updates should occur
- If the page reloads later, it should still show the correct stage based on timestamps
