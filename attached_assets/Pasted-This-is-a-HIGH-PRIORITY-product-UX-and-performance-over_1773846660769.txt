This is a HIGH-PRIORITY product, UX, and performance overhaul for the QuotePro “Sales Assistant” feature.

Right now this feature is not good enough.
Problems:
- responses take too long
- output formatting is weak and messy
- the result reads like a blob of text instead of a serious coaching tool
- it does not feel premium, actionable, or worthy of being inside a business app
- it should be a real AI sales coach for cleaning business owners, not a generic chatbot

The goal is to transform Sales Assistant into a fast, polished, high-value AI coaching tool that helps cleaning business owners:
- follow up on quotes
- handle objections
- structure sales process
- improve close rate
- decide what to say next
- get practical scripts, not generic essays

Do NOT make this a small styling patch.
This should be a real feature overhaul.

Do not break quoting, CRM, subscriptions, navigation, or other AI tools.
Before coding, first inspect the current implementation and identify:
- where the Sales Assistant screen/component lives
- what model/API/backend route it uses
- why the response is slow
- whether latency is caused by model choice, prompt size, backend timeout, or frontend rendering
- how the current prompt is structured
- how responses are parsed and rendered
- whether markdown / rich text is supported
- whether there is streaming support available
- whether response formatting is coming raw from model with no post-processing
- how feature gating works for this tool
Then implement carefully.

==================================================
PART 1 — FIX PERFORMANCE / SPEED
==================================================

Audit why responses are taking too long.

Possible issues to investigate:
- model too large / too slow for this use case
- excessive system prompt length
- bloated context being sent every time
- backend not streaming
- frontend waiting for full response before rendering
- slow network call / retry issue
- poor response parsing
- unnecessary conversation history being included
- serverless cold starts
- bad timeout handling

Fix the latency aggressively.

Preferred outcome:
- much faster first token / first visible response
- if possible, implement streaming so the user sees the answer building live
- if streaming is not feasible, at least improve perceived performance with better loading states and tighter prompts

Do not sacrifice answer quality unnecessarily, but prioritize responsiveness for this chat-like coaching tool.

==================================================
PART 2 — FIX RESPONSE FORMAT COMPLETELY
==================================================

The current output is unacceptable:
- giant paragraph blob
- hard to skim
- hard to use
- poorly formatted
- no separation of strategy vs. script vs. next steps

Redesign the response format.

The response should be rendered in a clear structured layout, not dumped as raw text.

Preferred structure for most answers:

1. Quick takeaway
- 1–3 short lines
- what the user should do

2. Recommended approach
- concise explanation
- why this is the right move

3. Ready-to-use script
- a clearly separated text/email/call script
- easy to copy

4. Alternate version(s)
- softer
- more direct
- premium tone
- recurring-focused version
as appropriate

5. Next step
- what to do if the customer does not respond
- what to send next
- when to follow up

This should render beautifully in cards/sections, not as one long chat bubble.
