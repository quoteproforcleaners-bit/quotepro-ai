GOAL
==================================================

The goal is to make the pricing engine:
- accurate
- deterministic

- correct for add-ons, deep cleans, and first-clean-only behavior

The pricing engine must follow the business rules exactly.

==================================================
KNOWN ISSUES TO FIX
==================================================

1. ADD-ONS ARE NOT CHANGING THE PRICE
Right now, when an add-on is selected, the quote price does not seem to change.
This must be fixed.

2. ADD-ONS SHOULD ONLY APPLY TO THE FIRST CLEAN
When add-ons are included, they should only affect the first clean price.
They should NOT continue being baked into recurring maintenance pricing unless explicitly intended by the business rules.

3. DEEP CLEAN SHOULD INCLUDE DEFAULT ADD-ONS
If the quote is a deep clean, it should include all add-ons by default EXCEPT inside fridge.

Important exception:
- inside fridge should NOT be automatically included in deep clean
- inside fridge should be reserved for move-in / move-out logic

4. WEB LOGIC MAY NOT MATCH IOS LOGIC
Audit and compare logic paths so the web quoting behavior matches the intended product pricing behavior.
If there is duplicated or diverging logic, identify it and either unify it or clearly align it.

5. USER PRICE SETTINGS MAY NOT BE RESPECTED
The pricing engine must follow the user’s price settings in the app.
It currently seems “very off,” which suggests settings may be ignored, partially applied, or overridden.

==================================================
PRICING RULES TO ENFORCE
==================================================

Please enforce these rules explicitly:

A. USER SETTINGS ARE SOURCE OF TRUTH
The user’s configured price settings must be the primary source of truth.
Do not silently override them with hardcoded defaults unless there is absolutely no data and a safe fallback is required.

B. ADD-ONS MUST CHANGE PRICE
Any selected add-on must visibly and actually increase the quote price according to the configured add-on pricing.

C. ADD-ONS ARE FIRST-CLEAN ONLY
Add-ons should only be included in the first clean total.
They should not be automatically rolled into future recurring pricing unless the product explicitly supports that and the user intended it.

D. DEEP CLEAN DEFAULTS
If cleaning type is deep clean:
- all standard deep-clean-related add-ons should be included by default
- EXCEPT inside fridge
- inside fridge should not be defaulted into deep clean pricing

E. MOVE-IN / MOVE-OUT LOGIC
Inside fridge should be reserved for move-in / move-out behavior, not automatically included in deep cleans.

F. RECURRING VS FIRST CLEAN
The engine must clearly distinguish:
- first clean / initial clean price
- recurring maintenance price

Add-ons and one-time extras should affect the first clean appropriately and not distort maintenance pricing.
