URGENT: Fix the QuotePro web app “Quote from Notes” / AI extraction -> quote creation flow.

Current problem:
When the software user pastes notes and clicks “Create Quote” / “Extract Details,” the resulting quote is junk.

Observed symptoms from the current flow:
- quote output contains broken pricing fields like NaN or $0
- pricing options show raw/internal keys like frequency, cleaningType, isDeepClean, isMoveInOut, isFirstTimeClean
- boolean/internal flags are rendered as price cards with nonsense values like $1
- extracted details are not being normalized into a clean quote structure
- add-ons and services are not being mapped cleanly
- the final quote detail screen looks broken and unprofessional
- the AI extraction does not produce a trustworthy quote-ready result

Treat this as a production-critical bug affecting the core promise of the app.

GOAL

Fix the “Quote from Notes” pipeline so that:
1. the AI extracts property/service details accurately from messy notes
2. extracted data is normalized into a proper structured quote object
3. internal flags are not rendered as customer-facing pricing options
4. the quote creation flow generates a clean, usable quote draft
5. pricing calculates correctly using the shared pricing engine
6. the quote detail page shows polished, human-friendly quote information
7. missing fields are flagged intelligently instead of producing junk output

THIS FEATURE SHOULD FEEL LIKE:
Paste messy notes -> AI extracts quote-ready details -> user reviews -> clean quote draft is created.

NOT:
Paste notes -> random internal fields -> NaN pricing -> broken quote object.

PRODUCT INTENT

The “Quote from Notes” feature should allow the software user to paste:
- walkthrough notes
- call notes
- text threads
- customer messages
- property descriptions
- voicemail transcriptions

And the system should intelligently extract:
- property details
- service type
- frequency
- condition
- pets
- add-ons
- move-in/move-out clues
- deep clean clues
- recurring cleaning clues
- missing-field warnings
- quote recommendations

Then create a clean draft quote that the user can review/edit/send.

ROOT CAUSE AREAS TO AUDIT

Audit the entire pipeline end to end:

1. notes input UI
2. AI extraction prompt / parsing logic
3. structured extraction schema
4. normalization / transformation layer
5. mapping into quote draft model
6. mapping into pricing engine inputs
7. quote draft save logic
8. quote detail rendering
9. pricing options rendering
10. customer-facing/internal field separation

I strongly suspect one or more of these is happening:
- AI extraction output schema is too loose
- raw extraction keys are being dumped directly into pricing options UI
- booleans are being coerced into price items
- missing values are becoming NaN
- extraction output is not normalized before quote creation
- quote detail page is rendering internal config keys instead of human-facing services/options
- the quote creation flow is mixing:
  - extracted attributes
  - pricing settings
  - pricing options
  - service flags
  incorrectly

You must fix the root cause, not just hide broken fields in the UI.

REQUIRED FIXES

1. DEFINE A STRICT EXTRACTION SCHEMA

Create or fix a structured extraction schema for AI output.
The extraction step should produce a typed object like:

{
  property: {
    homeType,
    squareFootage,
    bedrooms,
    bathrooms,
    halfBathrooms,
    residents
  },
  service: {
    cleaningType,
    frequency,
    condition,
    isDeepClean,
    isMoveInOut,
    isFirstTimeClean
  },
  pets: {
    hasPets,
    petTypes,
    sheds
  },
  addOns: [
    ...
  ],
  customerSignals: {
    urgency,
    budgetSensitivity,
    requestedByDate
  },
  missingFields: [
    ...
  ],
  confidence: ...,
  notesSummary: ...
}

Use a clean, stable schema.
Do not let arbitrary raw LLM text flow directly into quote creation.

2. NORMALIZATION LAYER

Create a normalization / transformation layer between AI extraction and quote creation.

This layer should:
- validate extracted values
- coerce types correctly
- convert synonyms to known enum values
- map messy phrasing to supported quote fields
- prevent invalid/null/undefined/NaN values
- separate internal flags from customer-facing options
- compute safe defaults where appropriate
- produce a quote-ready draft payload

Examples:
- “biweekly” -> frequency = biweekly
- “every 2 weeks” -> frequency = biweekly
- “move out” / “move-out” -> isMoveInOut = true
- “very dirty” -> condition = heavy or very_dirty based on supported enums
- “inside oven and fridge” -> addOns = [inside_oven, inside_fridge]

3. DO NOT RENDER INTERNAL FIELDS AS PRICING OPTIONS

This is critical.

Fields like:
- frequency
- cleaningType
- isDeepClean
- isMoveInOut
- isFirstTimeClean

should NOT appear as raw pricing option cards with values like:
- frequency $NaN
- isDeepClean $1

These are internal quote attributes or modifiers, not customer-facing package cards.

Fix the rendering logic so the quote detail page displays:
- service summary
- pricing breakdown
- selected add-ons
- service modifiers
- calculated total
- package tiers if applicable

But NOT raw internal keys as standalone pricing options.

4. FIX NaN / $0 / BAD PRICING OUTPUT

Any quote created from notes must pass through the same shared pricing engine used elsewhere in the app.

Requirements:
- extracted quote data must map cleanly into pricing inputs
- invalid numeric values must be blocked or defaulted safely
- no NaN should ever be rendered
- no $0 final quote should appear unless the quote is truly intended to be zero
- missing pricing inputs should create review warnings, not broken output

5. REVIEW STEP BEFORE FINAL CREATE

Improve the UX so “Quote from Notes” is not blindly auto-creating junk quotes.

Recommended flow:
1. user pastes notes
2. AI extracts structured details
3. user reviews extracted fields in a clean review panel
4. user edits/fills any missing info
5. then user clicks Create Quote
6. quote draft is created cleanly

If the current flow auto-creates immediately, change it to a review-first flow if needed.

This is especially important for trust.

6. IMPROVE AI EXTRACTION QUALITY

The AI extraction should be better at pulling:
- bedrooms
- bathrooms
- square footage
- home type
- condition
- service frequency
- add-ons
- pets
- special requests
- urgency
- move-in/move-out context
- deep clean / first-time clean clues

Examples of inputs it should handle well:
- “3 bed, 2 bath home in Malvern. First-time clean, 1 dog, customer wants biweekly if price makes sense.”
- “Move-out clean for 2,100 sq ft condo. Needs inside oven and fridge. Very dirty kitchen.”
- “Customer texted: standard clean every 2 weeks, 4 bed, 3 bath, 2 cats, wants quote by Friday.”

The extracted output should be clean, structured, and quote-ready.

7. MISSING FIELD STRATEGY

When critical info is missing, do NOT produce junk.

Instead:
- flag missing fields clearly
- allow quote creation with partial info only if the product supports safe defaults
- surface what still needs review

Examples:
- missing square footage
- missing bed/bath counts
- ambiguous frequency
- unclear service type

Display:
- “AI extracted most details, but square footage is missing.”
- “Review recommended before creating quote.”

8. CLEAN QUOTE DETAIL PAGE OUTPUT

Once a quote is created from notes, the quote detail page should look polished.

It should show sections like:
- Property Details
- Service Summary
- Add-Ons
- Pricing Breakdown
- Quote Total
- AI Notes Summary / Source Notes if useful internally
- Missing info / review flags if relevant

It should NOT show:
- raw JSON-ish field names
- booleans as price line items
- undefined/null/NaN
- internal modifier flags as customer-facing options

9. HUMAN-FRIENDLY LABELS

Normalize any internal keys to proper labels.
Examples:
- inside_oven -> Inside Oven
- inside_fridge -> Inside Fridge
- isMoveInOut -> Move-In / Move-Out
- isFirstTimeClean -> First-Time Clean
- cleaningType -> Cleaning Type
- frequency -> Frequency

But again: not every internal field belongs in the pricing options section.

10. PRICING INTEGRATION

The extracted quote must plug into the real quote/pricing system correctly.

Use the shared pricing engine.
If certain extracted attributes influence pricing, apply them properly:
- square footage
- bedrooms
- bathrooms
- service type
- condition
- add-ons
- frequency
- deep clean / move-out modifiers
- first-time clean modifiers if supported

Never let the AI directly invent final totals.
AI extracts structured data; pricing engine calculates the price.

- left side: raw notes input
