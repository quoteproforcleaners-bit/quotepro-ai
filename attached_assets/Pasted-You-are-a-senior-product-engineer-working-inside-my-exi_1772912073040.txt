FEATURE 4: IMPORT JOBBER CLIENTS
--------------------------------------------------

Build a first version of “Import Jobber Clients”.

GOAL
Allow a connected user to import existing Jobber clients into QuotePro so they can create quotes faster.

SCOPE FOR V1
- only import core client data
- no overengineering
- make it safe and understandable

UI
Add an “Import from Jobber” action in an appropriate place such as:
- CRM / Customers page
- Contacts page
- or Integration settings screen

Flow:
1. user clicks “Import Jobber Clients”
2. system fetches Jobber clients
3. show simple review/import UI:
   - client name
   - email
   - phone
   - address
4. user can:
   - import all
   - import selected
5. show results summary:
   - imported count
   - skipped duplicates
   - failed count if any

DATA HANDLING
Map Jobber client data into QuotePro customer/contact records.

At minimum import:
- full name / display name
- email
- phone
- address if available
- Jobber external ID for future sync reference

DEDUPLICATION
Implement safe duplicate handling.
Before creating a new QuotePro customer, attempt to detect duplicates using a reasonable strategy such as:
- existing external Jobber ID match
- email match
- phone match
- fallback name/address similarity if already supported

If duplicate found:
- skip duplicate
- mark as skipped
- do not create messy duplicate records

TECHNICAL
Create a dedicated import service module if needed.
Paginate if Jobber clients can be large.
Handle token refresh automatically using the existing Jobber integration layer.

DOCUMENT LIMITATIONS
If full pagination/import-all is too large for one pass, implement a sensible first page / batch import with a clean structure that can be expanded later.
