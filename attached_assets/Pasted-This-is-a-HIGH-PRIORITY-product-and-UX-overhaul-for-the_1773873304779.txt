This is a HIGH-PRIORITY product and UX overhaul for the QuotePro web dashboard.

The dashboard currently looks clean and trustworthy, but it is too passive and too “reporting-oriented.”
It shows data, but it does not actively drive user behavior, activation, or revenue.

I want to transform it into a revenue-driving dashboard that feels like:
- a sales control center
- a coaching dashboard
- a daily action hub
- a business growth command center for cleaning business owners

Do NOT break existing data, auth, routing, sidebar nav, subscription logic, or dashboard cards unless they are intentionally being improved.
Before coding, first inspect the current dashboard implementation and identify:
- where the dashboard page/component lives
- how dashboard metrics are fetched
- how “Revenue Protection Score” is calculated
- whether there is onboarding state already available
- whether there is user setup state available (pricing configured, first quote created, etc.)
- whether there is task/follow-up data available
- which cards are real vs placeholder/empty-state driven
Then implement carefully.

==================================================
CORE PRODUCT DIRECTION
==================================================

The dashboard should no longer feel like:
- a static reporting page
- a blank admin panel
- a passive summary screen

It should feel like:
- “Here’s what to do next to make money”
- “Here’s what is blocking your growth”
- “Here’s where revenue is at risk”
- “Here’s the next best action”

The dashboard should drive activation and behavior.

==================================================
PRIMARY GOALS
==================================================

1. Replace passive empty/neutral states with action-oriented revenue states
2. Add clearer next steps for new and low-activity users
3. Surface AI and growth tools more prominently
4. Turn the dashboard into a daily operating system, not just analytics
5. Make the user feel progress, urgency, and opportunity

==================================================
PART 1 — REPLACE PASSIVE HERO / TOP STATE
==================================================

The current top section (“All Caught Up” / Revenue Protection Score area) is too passive, especially for low-activity users.

I want the top section to become dynamic and behavior-driving.

The hero/top state should adapt based on user context, for example:

A. New user / zero quotes:
- “Create your first quote and start building pipeline”
- “Your first $1,000 starts with one quote”
- CTA: Create First Quote
- Secondary CTA: Configure Pricing

B. User has pricing incomplete:
- “Your pricing setup is incomplete”
- “Finish pricing settings so QuotePro can generate accurate quotes”
- CTA: Finish Pricing Setup

C. User has quotes sent but no follow-up:
- “Revenue is sitting in your pipeline”
- “You have quotes that need follow-up”
- CTA: Review Follow-Ups

D. User is active and healthy:
- show positive momentum / score / progress
- but still give a next best action

Use the Revenue Protection Score if it is real and useful, but make the card more action-driven and contextual.

==================================================
PART 2 — ADD A “TODAY’S REVENUE MOVES” PANEL
==================================================

Add a highly visible section near the top called something like:
- Today’s Revenue Moves
- Next Best Actions
- Revenue Opportunities Today

This section should tell the user what actions to take right now.

Examples:
- Create your first quote
- Follow up with 2 leads
- Send an intake form
- Review quotes pending > 48 hours
- Ask for 1 review
- Re-engage a lost opportunity
- Complete pricing setup

Each action item should ideally include:
- short explanation
- urgency/opportunity framing
- CTA button
- maybe estimated impact if data supports it

This section is one of the most important changes.

==================================================
PART 3 — MAKE EMPTY STATES SELL ACTION
==================================================

Right now many cards likely show zeros and feel dead.

I do NOT want dead dashboards.

For cards with zero/no data, replace sterile empty states with action-driven copy.

Examples:

Instead of:
- Total Quotes: 0

Use:
- “No quotes sent yet”
- “Create your first quote to start building revenue”

Instead of:
- Revenue Won: $0

Use:
- “No revenue won yet”
- “The fastest way to revenue is sending your first quote”

Instead of:
- Close Rate: 0%

Use:
- “No quote activity yet”
- “Send quotes and follow up to start tracking close rate”

This should feel motivating, not depressing.

==================================================
PART 4 — SURFACE AI AS A VALUE DRIVER
==================================================

Right now AI appears to be buried in the sidebar.
That is a mistake because AI is one of the most differentiating parts of QuotePro.

I want AI surfaced on the dashboard more clearly.

Add a section/card such as:
- AI Growth Tools
- AI Revenue Assist
- AI Sales Assist

This area should highlight useful revenue-driving AI actions like:
- Handle objections
- Draft follow-up message
- Turn notes into quote
- Generate recurring upsell script
- Build a customer reply
- Extract quote details from notes

This should make the dashboard feel more special and more valuable.

==================================================
PART 5 — ADD A STRONG “START HERE” STATE FOR NEW USERS
==================================================

If the user is new or low-setup, the dashboard should explicitly guide them.

Add a “Start Here” checklist / guided setup block near the top if appropriate.

Suggested tasks:
1. Configure pricing
2. Create your first quote
3. Send intake form or enable quote requests
4. Add your first customer
5. Turn on follow-ups or growth tools

This checklist should:
- feel clear
- show progress
- disappear or collapse when complete
- help activation instead of overwhelming

If there is already onboarding/progress state available, reuse it.

==================================================
PART 6 — MAKE THE DASHBOARD MORE COACHING-ORIENTED
==================================================

I want the dashboard to feel like it is coaching the user, not just reporting numbers.

Possible coaching examples:
- “Quotes sitting longer than 3 days are less likely to close. Review follow-ups.”
- “Recurring customers increase lifetime value. Use AI to position recurring.”
- “You’re one quote away from building momentum this week.”
- “Most cleaning businesses win more jobs when they follow up within 48 hours.”

Use light-touch coaching copy throughout key areas, especially when data is sparse.

==================================================
PART 7 — IMPROVE THE FOLLOW-UP / STREAK / MOMENTUM AREAS
==================================================

The current streak / momentum widgets feel under-leveraged.

Improve them so they are more motivating and tied to business outcomes.

Examples:
- connect streaks to close-rate behavior
- connect momentum to daily actions
- add encouraging but revenue-focused copy
- make it obvious why the streak matters

This should feel more like a performance engine, less like a novelty widget.

==================================================
PART 8 — BETTER INFORMATION HIERARCHY
==================================================

Audit the current layout and improve the order of information.

Recommended general hierarchy:
1. Dynamic hero / top revenue state
2. Today’s Revenue Moves / Next Best Actions
3. Core KPI cards
4. Sales momentum / close rate / pipeline health
5. AI growth tools
6. Follow-up / streak / weekly recap
7. deeper analytics and charts

I want the dashboard to first answer:
- What should I do next?
Then:
- How is the business doing?

Not the other way around.

==================================================
PART 9 — KEEP IT CLEAN AND PREMIUM
==================================================

Do NOT make the dashboard noisy or cluttered.

The redesign should:
- stay modern and premium
- preserve the clean trustworthiness of the current UI
- add urgency and action without becoming cheesy
- use whitespace well
- maintain strong hierarchy
- not overload with too many cards

Think:
“premium SaaS growth dashboard for cleaning businesses”
not
“busy generic CRM”

==================================================
PART 10 — CTA STRATEGY
==================================================

I want clearer CTA strategy across the dashboard.

Primary CTA should likely remain:
- New Quote
but potentially with stronger surrounding support

Also introduce contextual CTAs like:
- Create First Quote
- Finish Pricing Setup
- Send Intake Form
- Review Follow-Ups
- Use AI Assistant
- Turn Notes Into Quote

Buttons should route users into the most important workflows quickly.

==================================================
PART 11 — DATA-AWARE STATES
==================================================

Where possible, make dashboard modules dynamic based on real user state, such as:
- has sent quotes?
- has configured pricing?
- has any customers?
- has follow-ups due?
- has reviews?
- has recurring jobs?
- has AI tools unused?
- has lead capture configured?

The dashboard should adapt to user maturity instead of showing the same layout to everyone.

==================================================
PART 12 — GROWTH / REVENUE FRAMING
==================================================

Use stronger product language throughout the dashboard.

The dashboard should reinforce themes like:
- pipeline
- momentum
- revenue risk
- next best action
- close rate
- follow-up timing
- recurring growth
- quote velocity
- sales discipline

This language should make the product feel more outcome-driven.

==================================================
PART 13 — VALIDATION / TESTING
==================================================

After implementation, validate:
1. New user sees clear “start here” / first action guidance
2. Low-activity user sees action-oriented empty states
3. Active user sees meaningful next actions, not dead space
4. Dashboard still feels clean and not cluttered
5. AI tools are surfaced more visibly
6. CTA buttons route correctly
7. Existing KPI data still renders correctly
8. No broken fetches or loading states
9. Responsive behavior still works
10. Revenue-driving messaging feels helpful, not gimmicky
