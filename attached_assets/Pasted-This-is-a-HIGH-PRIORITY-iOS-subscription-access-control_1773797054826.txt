This is a HIGH-PRIORITY iOS subscription access-control audit for QuotePro.

There is a serious monetization bug:
No matter which tier I choose in the iOS app, I appear to have access to everything.

This must be fixed immediately.

The goal is to audit the entire entitlement / feature-gating system and make sure users only get access to features that belong to their subscription tier.

Do NOT treat this as a cosmetic UI fix.
This is a real access-control and monetization enforcement issue.

Important:
Do not break auth, quoting, CRM, onboarding, or purchase flows.
Before coding, first inspect the current iOS app architecture and identify:
- where RevenueCat is initialized
- where purchases are completed
- where entitlements are read
- where plan tier is stored in app state
- where feature access is currently checked (if anywhere)
- where quote builder, AI quoting, CRM, follow-ups, upsells, dashboards, automations, integrations, and commercial quoting features are triggered
- whether there is any fallback logic that incorrectly defaults users to full access
- whether tier state is cached incorrectly
- whether paywall selection UI is disconnected from actual entitlement enforcement

Then implement carefully.

==================================================
PRIMARY GOAL
==================================================

Create a real, centralized feature-gating system for iOS so that:

- Starter users only get Starter features
- Growth users only get Growth + Starter features
- Pro users get everything
- Free users, if applicable, get only free access

Users should be able to SEE higher-tier features in many places, but if they are not entitled:
- they should not be able to use them
- they should be blocked cleanly
- they should get a contextual upgrade prompt

In many cases, greying out locked features is preferred so users understand what exists and what they unlock by upgrading.

==================================================
KNOWN BUG
==================================================

Currently, users can access everything regardless of selected/purchased tier.

Possible causes to audit:
- all tiers mapped to same access state
- RevenueCat entitlement not being read correctly
- fallback state defaulting to highest access
- UI tier selection being mistaken for actual entitlement
- no real access checks in feature entry points
- components rendering as unlocked by default
- missing centralized access-control system
- purchase success path not refreshing entitlement state
- old legacy “pro access” logic still hanging around

You must fully audit and fix this.

==================================================
PART 3 — REAL FEATURE ENFORCEMENT
==================================================

Do NOT only hide UI.
Actually block access.

For every premium feature:
- block the action if the user is not entitled
- prevent navigation if needed
- prevent API-triggering actions if needed
- show upgrade modal / upgrade prompt

Examples:
- Starter taps AI quote builder → blocked + Growth upsell
- Starter taps smart upsells → blocked + Growth upsell
- Starter taps follow-up automation → blocked + Growth upsell
- Growth taps commercial quoting → blocked + Pro upsell
- Growth taps integrations → blocked + Pro upsell

Do not let users trigger premium actions just because a button is visible.

- greyed out buttons/cards/modules
- lock icon or subtle “Upgrade” chip
- reduced opacity
- non-clickable or intercept click/tap with upgrade modal
- concise explanation of which plan unlocks it

This should be done in a way that:
