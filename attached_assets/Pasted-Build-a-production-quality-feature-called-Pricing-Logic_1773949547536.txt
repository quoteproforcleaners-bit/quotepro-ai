Build a production-quality feature called “Pricing Logic Engine” for QuotePro AI, available in both the iOS app and the web app.

Context:
QuotePro AI helps cleaning businesses create quotes faster and more consistently. This new feature should let a user upload a small set of past quotes/jobs, answer a setup questionnaire about how they price, and then use AI to infer their pricing logic, identify revenue leaks, suggest pricing improvements, and generate a reusable pricing rules engine that can be applied across the product.

The feature must work for:
1. Web app
2. iOS app

The architecture should support:
- shared backend business logic
- shared database schema
- consistent rules engine behavior across platforms
- platform-specific UI for web and iOS
- extensibility for future quote types and pricing models

Primary outcome:
A cleaning business owner uploads 3–20 past jobs or quotes, fills out a pricing setup flow, and the system outputs:
- inferred current pricing logic
- inconsistencies / underpricing observations
- suggested revenue improvements
- editable pricing rules
- one-click publish of approved pricing logic into the quote-generation workflow

Build this as a serious SaaS feature, not a toy demo.

Product requirements:

FEATURE NAME
Pricing Logic Engine

TARGET USERS
- Solo cleaning business owners
- Residential cleaning companies
- Commercial cleaning companies
- Multi-estimator teams that need consistent quoting

- Standard clean

3. Pricing Setup Questionnaire
Create a guided form that captures business assumptions such as:
- minimum job price
- service area / zip codes
- target hourly revenue
- preferred crew size assumptions
- supplies included or not
- recurring discounts
- deep clean multiplier
- move-out multiplier
- pet surcharge
- travel surcharge
- add-on pricing
- “never go below” pricing thresholds
- whether pricing varies by home condition
- whether pricing varies by frequency
- whether pricing varies by square footage or bed/bath count

4. AI Analysis
After upload + questionnaire, run an AI analysis pipeline that:
- normalizes imported jobs
- infers likely pricing patterns
- identifies inconsistencies
- identifies likely underpriced situations
- suggests pricing improvements
- produces a draft pricing model

The analysis should generate the following sections:

A. “How You Appear to Price Today”
Examples:
- minimum observed price
- price-per-square-foot trends
- bedroom/bathroom weighting
- deep clean uplift patterns
- recurring discount patterns
- add-on pricing patterns
- service-area / travel effects
- underused surcharges

B. “Potential Revenue Opportunities”
Examples:
- deep cleans may be underpriced
- extra bathrooms may not be weighted enough
- pet homes may not be monetized
- heavy-condition jobs may be priced like standard jobs
- minimum price floor may be too low

C. “Recommended Pricing Rules”
Generate editable pricing rules in structured form.

5. Pricing Rules Editor
Create an editable rules engine UI where users can review and modify rules before publishing.

Support rule types such as:
- base minimum price
- base price by service type
- square footage range pricing
- bedroom adjustments
- bathroom adjustments
- half-bath adjustments
- condition multipliers
- frequency discounts
- pet surcharge
- add-on fixed prices
- zip code / travel surcharge
- estimated hours fallback
- crew-size adjustments
- first-time clean multiplier

Every rule should include:
- label
- type
- input variables used
- formula or pricing value
- explanation
- source = inferred / user-created / AI-recommended
- active/inactive status

6. Publish Pricing Logic
Once approved, users can publish pricing logic.
Publishing should make the pricing logic available to the quote-generation workflow across both web and iOS.

7. Quote Calculator Integration
Build a basic quote calculator interface that uses the published pricing logic.
A user should be able to enter a new job and get:
- recommended quote
- price breakdown
- triggered rules
- explanation of why the quote is what it is

8. Explainability
This is very important.
The system must explain recommendations in plain English.
Do not present black-box outputs.
Every AI recommendation should show:
- why it was suggested
- what data it was based on
- confidence or strength level
- what would change if accepted

9. Human trust / approval
The AI should never silently overwrite pricing.
Users must review and approve rules before publishing.
