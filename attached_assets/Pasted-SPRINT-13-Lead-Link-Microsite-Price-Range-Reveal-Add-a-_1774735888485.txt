SPRINT 13 — Lead Link Microsite Price Range Reveal

Add a progressive disclosure flow to the Lead Link 
microsite that shows a live price range estimate after 
the first 3 fields, then gates the full quote behind 
contact info submission. This is the highest-leverage 
conversion rate improvement in the entire quote flow.

TECH STACK: React 18, existing Lead Link microsite 
component, existing pricing calculation logic

Submits everything →

PRICE RANGE CALCULATION
═══════════════════════════════

When user completes Step 1, calculate a price range 
client-side using the software user's configured 
pricing settings (already available via the Lead Link 
API — include baseRate, minimumTicket, serviceTypes 
in the Lead Link config payload).

RANGE LOGIC:
Use existing calculatePricingRecommendations() logic 
or a simplified version:

baseEstimate = calculateBasePrice(bedrooms, bathrooms, 
  cleaningType, softwareUserPricingSettings)

lowEstimate = Math.max(
  softwareUserSettings.minimumTicket,
  baseEstimate * 0.85
)

highEstimate = baseEstimate * 1.25

Round both to nearest $5:
  lowEstimate = Math.round(lowEstimate / 5) * 5
  highEstimate = Math.round(highEstimate / 5) * 5

═══════════════════════════════
MOBILE OPTIMIZATION
═══════════════════════════════

The Lead Link microsite is primarily visited 
on mobile. Enforce these mobile-first rules:

- Max width: 480px, centered on desktop
- All tap targets: minimum 48px height
- Segmented selectors: wrap to 2 rows on 
  screens < 360px wide
- Font sizes: minimum 16px on all inputs 
  (prevents iOS zoom on focus)
- Sticky CTA button at bottom of screen 
  on Step 2 and Step 3:
  Fixed position, full width, 
  background matches page (no transparency)
  Safe area padding on iPhone notch devices

- Keyboard behavior:
  When a text input is focused, scroll the 
  page so the input is visible above the keyboard
  Use ScrollView with keyboardShouldPersistTaps
