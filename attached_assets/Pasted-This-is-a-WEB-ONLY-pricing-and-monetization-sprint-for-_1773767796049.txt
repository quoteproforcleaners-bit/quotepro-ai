This is a WEB-ONLY pricing and monetization sprint for QuotePro.

Do NOT work on iOS in this pass.
Do NOT touch RevenueCat or native mobile subscription flows yet.
Focus only on the web app and web pricing experience.

Important:
Do not break the existing web app.
Do not overwrite core quote, CRM, auth, or dashboard functionality.
This must integrate cleanly into the current QuotePro web codebase.
Before coding, first inspect the existing architecture and identify:
- current pricing page (if any)
- auth/signup flow
- trial flow
- billing/subscription models
- Stripe/web billing integration if present
- dashboard gating / upgrade prompts
- feature flags / entitlement logic
- any plan-related database schema

Then implement carefully.

==================================================
GOAL
==================================================

QuotePro is currently underpriced and under-structured on web.

We need a pricing strategy that:
- increases perceived value
- pushes users toward the best revenue plan
- supports annual billing
- reduces churn
- improves LTV
- uses clear Good / Better / Best pricing logic
- matches the product promise:
  Quote faster. Close more jobs. Increase revenue.

This is not a generic pricing page.
This should feel intentional, premium, and conversion-aware.

==================================================
STRATEGY TO IMPLEMENT
==================================================

Build a 3-tier pricing structure for the WEB APP only:

1. STARTER
2. GROWTH (recommended / default emphasis)
3. PRO

Use this pricing model:

----------------------------------------
STARTER
----------------------------------------
$19/month

Purpose:
Low-friction entry plan for solo operators who want to try the product.

Include:
- limited monthly quotes (suggest 20/month unless current architecture makes a different limit easier)
- branded intake forms
- basic quote creation
- basic Good / Better / Best quoting
- basic CRM / lead capture
- email support

Do NOT include:
- advanced AI tools
- advanced automations
- advanced upsell intelligence
- advanced reporting / revenue insights

----------------------------------------
GROWTH (MOST IMPORTANT)
----------------------------------------
$49/month
or
$490/year

This should be the featured plan and the one we push hardest.

Include:
- unlimited quotes
- AI quote builder
- smart upsell recommendations
- follow-up tools
- CRM
- quote requests / public lead capture
- better automations
- revenue-focused workflow features
- anything that represents the real core value of QuotePro

Annual pricing:
- $490/year
- clearly frame as “2 months free” or equivalent savings
- annual should be visually emphasized as the best value

Trial:
- add a 7-day free trial on Growth plan for web
- trial should be clearly stated
- if there is a safe way to do this in existing billing logic, implement it
- if trial infrastructure already exists, reuse it
- if not, add the web-facing hooks and structure cleanly

----------------------------------------
PRO
----------------------------------------
$99/month
or
$990/year

Purpose:
Anchor value and capture larger / more serious operators.

Include everything in Growth plus:
- advanced automation
- advanced reporting / revenue intelligence
- priority support
- future-facing premium capabilities if current architecture supports placeholders
- anything already in the product that represents higher-value usage
- if some premium features are not yet fully built, label carefully and honestly

This plan should feel premium and serious, but not be the main default.

Build or redesign the web pricing page so it feels polished, premium, and intentional.

- headline focused on outcome, not generic SaaS language

- 3 pricing cards
- GROWTH visually marked as:
  - Most Popular
  - Best Value

- FAQ section below

Suggested messaging direction:
Do NOT use vague startup copy.

Examples:

==================================================
CONVERSION REQUIREMENTS
==================================================

The pricing page should intentionally push users toward Growth.

Implement the following:
- make Growth card slightly larger or more visually prominent
- emphasize annual savings most strongly on Growth
- use stronger CTA on Growth like:
  - Start free trial
  - Start 7-day trial
  - Start growing revenue
- Starter CTA can be softer:
  - Get started
- Pro CTA can be:
  - Contact sales if needed
  - Upgrade to Pro

If appropriate, add a line under Growth like:
- Best for most cleaning businesses
- Most users choose this plan
- Best balance of speed + revenue tools
