You are a senior product engineer, conversion-focused SaaS designer, and full-stack developer working inside my existing QuotePro AI codebase.

GOAL
Build a production-ready new feature called:

“Instant Quote Page”

This feature should turn a normal cleaning quote into a beautiful, customer-facing quote experience that helps the cleaning company close more jobs, present upsells professionally, optionally collect a deposit, create urgency, and allow the customer to accept/book easily.

This should become one of the strongest differentiators in QuotePro.

CORE PRODUCT STRATEGY
This is not just a static estimate screen.
This is a conversion-focused sales page for each quote.

It should feel:
- modern
- premium
- simple
- trustworthy
- mobile-first
- high-converting
- easy for customers to understand and act on

It should support:
- beautiful quote presentation
- Good / Better / Best options when available
- live add-on selection
- optional deposit collection
- optional quote expiration countdown / urgency
- AI-powered follow-up if quote is not accepted
- optional customer reviews / trust block
- acceptance flow that ties into the existing QuotePro quote acceptance workflow
- compatibility with existing Jobber sync after acceptance

==================================================
FEATURE OVERVIEW
==================================================

When a QuotePro user creates and sends a quote, they should have the option to generate a unique customer-facing quote page URL.

Example:
- /quote/[publicQuoteId]

The customer opens the link and sees:
1. company branding
2. service summary
3. pricing options (Good / Better / Best if available)
4. optional add-ons that can be toggled live
5. updated total price
6. optional urgency/expiration messaging
7. optional trust/testimonial section
8. preferred scheduling/request section
9. optional deposit collection
10. accept quote button
11. confirmation state after acceptance

If Jobber integration is connected, accepted quotes should continue to follow the existing Jobber sync workflow.

==================================================
1. PUBLIC QUOTE PAGE
==================================================

Build a public-facing quote page route.

Example route:
- /quote/[publicId]

This page must be:
- secure enough for public sharing
- readable on mobile
- visually polished
- fast loading
- easy to understand for homeowners and business buyers
- designed like a mini sales page, not a spreadsheet

PAGE CONTENT

A. COMPANY HEADER
Display:
- company name
- logo if available
- optional phone/email
- optional “Licensed / Insured” or trust note if this already exists in account settings
- trust-building visual treatment

B. QUOTE TITLE / INTRO
Examples:
- “Your Cleaning Quote”
- “Your Personalized Cleaning Quote”
Subtext:
- “Review your options and accept when ready.”

C. SERVICE SUMMARY CARD
Include:
- service type
- square footage if available
- bedrooms / bathrooms if available
- property type if available
- frequency if available
- notes relevant to scope

D. PRICING OPTIONS SECTION
Support a 3-tier quote layout when available:
- Good
- Better
- Best

Each option should appear as a visually distinct selectable card.

Each card should include:
- plan/option title
- short description
- included services summary
- base price
- optional “Most Popular” badge on the middle option if configured

If the quote only has one option, gracefully render a single-option version.

E. ADD-ONS SECTION
If add-ons exist, show them as selectable toggles or checkboxes.

Examples:
- Inside Oven
- Inside Fridge
- Interior Windows
- Laundry
- Bed Linen Change
- etc.

When customer toggles add-ons:
- total price should update live
- selected quote option total should recalculate immediately
- UI should remain very smooth and understandable

F. TOTAL / SUMMARY PANEL
Sticky on desktop if appropriate.
Clearly show:
- selected package
- selected add-ons
- subtotal
- deposit amount if required
- remaining balance if deposit applies
- total price
- any one-time / recurring notes if applicable

G. TRUST / SOCIAL PROOF BLOCK
Add an optional section on the quote page for trust-building.

If account settings or quote settings support it, show:
- 1–3 short customer testimonials
- average review score if available
- short trust statement like:
  “Trusted by homeowners across [service area]”
- optional Google review count if the app already stores this
- optional “background-checked cleaners” or “insured & bonded” copy if the business has configured it

This section should be tasteful and concise.
Do not clutter the page.

H. URGENCY / EXPIRATION BLOCK
Support optional quote expiration logic.

Examples:
- “This quote is valid through [date]”
- “Pricing expires in 48 hours”
- subtle countdown treatment if expiration is short-term and appropriate

Rules:
- must be configurable by the QuotePro user
- should create urgency without feeling spammy
- expired quotes should show a graceful state:
  “This quote has expired. Please contact us for an updated quote.”

I. SCHEDULING / REQUEST SECTION
For V1, allow:
- preferred date input
- preferred time window dropdown or text field
- optional notes/comments field

This should capture customer preference, not full dispatching unless already supported.

J. DEPOSIT COLLECTION SECTION
Support optional deposit collection as part of quote acceptance.

If deposit is enabled for the quote, show:
- deposit required amount
- what the deposit secures
- remaining balance due later

Examples:
- flat deposit amount
- percentage-based deposit

If existing payment infrastructure already exists in the app:
- integrate with it cleanly

If no payment infrastructure exists yet:
- build the UI/data model in a modular way and leave the payment step abstracted behind a payment provider integration point
- if feasible, implement with the app’s current payment stack only if it already exists and is safe

Primary objective:
- allow optional deposit requirement to be tied to acceptance flow
- acceptance flow should clearly distinguish between:
  - accept without deposit
  - accept with deposit required

K. ACCEPT CTA
Primary CTA button:
- “Accept Quote”
Alternative if deposit is enabled:
- “Accept & Pay Deposit”
or
- “Approve Quote”

When customer clicks accept:
- validate required fields
- validate expiration
- validate payment if deposit is mandatory
- show loading state
- persist selected option
- persist selected add-ons
- persist preferred date/time
- persist notes
- persist deposit/payment status if applicable
- mark quote as accepted
- trigger existing acceptance flow in QuotePro
- trigger existing Jobber sync flow if enabled
- show success state

L. SUCCESS STATE
After acceptance show a beautiful confirmation state.

Examples:
Headline:
- “Quote Accepted”
- “You’re All Set”

Body:
- thank you message
- selected package summary
- selected add-ons
- preferred date/time submitted
- deposit received status if applicable
- note that company will follow up to confirm appointment if needed

If the system already supports email/text confirmation, hook into it if safe.
If not, just show a polished on-screen confirmation.

9. AI FOLLOW-UP IF QUOTE NOT ACCEPTED
==================================================

Build a first version of automated follow-up for unaccepted quotes.

GOAL
If a quote page is viewed but not accepted, or sent but not accepted after a chosen time period, QuotePro should help the user follow up automatically.

For V1, support:
- enable/disable follow-up per quote or account setting
- configurable trigger such as:
  - 24 hours after sending
  - 24 hours after first view
  - 48 hours after send
- generate an AI-powered follow-up message draft using quote context

Examples:
“Hi Sarah, just checking in on the cleaning quote we sent over. Let me know if you’d like to move forward or if you have any questions.”

Requirements:
- use existing AI messaging stack if already present
- if email/SMS send infrastructure exists, integrate with it
- if not, generate a ready-to-send draft inside the app
- keep messaging friendly and natural, not spammy

Track status:
- follow-up pending
- follow-up sent
- follow-up skipped
- quote already accepted, so cancel follow-up

If there is already a scheduler/automation system in the app, reuse it.
If not, build the cleanest lightweight version possible.
