BACKEND — REMINDER SETTINGS
═══════════════════════════════

1. ADD to user preferences/settings table:
   customerEmailReminderEnabled: boolean
   customerEmailReminderDays: integer (1, 2, 3, 7)
     null = disabled
   customerSmsReminderEnabled: boolean
   customerSmsReminderDays: integer (0, 1, 2, 3)
     0 = same day, null = disabled

2. SAVE onboarding selections:
   POST /api/user/reminder-preferences
   Body: {
     emailReminderDays: number | null,
     smsReminderDays: number | null
   }

3. CREATE reminder scheduler 
   /jobs/appointmentReminderScheduler.ts:
   
   Runs every hour (cron: 0 * * * *)
   
   For EMAIL reminders:
     Find all jobs where:
       startDatetime = NOW + emailReminderDays days
       (within a 1-hour window of the scheduled time)
       customer has email
       reminder not already sent 
       (check reminder_log table)
       customerEmailReminderEnabled = true
     
     Send email with:
       Subject: "Your cleaning appointment 
       is [tomorrow / in X days]"
       
       Body:
       "Hi [FirstName],
       
       This is a reminder that [BusinessName] 
       is scheduled to clean your home on 
       [Day], [Date] at [Time].
       
       If you need to reschedule or cancel, 
       please reply to this message or call 
       us at [BusinessPhone].
       
       See you [tomorrow / on Day]!
       [SenderName]
       [BusinessName]"

   For SMS reminders:
     Same query but for SMS
     Message:
     "Hi [FirstName]! Reminder: 
     [BusinessName] is cleaning your home 
     [tomorrow / on Day] at [Time]. 
     To reschedule call [Phone]. 
     Reply STOP to opt out."
     
     Character limit: keep under 160 chars

4. CREATE reminder_log table:
   id, job_id, reminder_type (email|sms), 
   sent_at, status (sent|failed), 
   error_message

5. PREVENT DUPLICATE SENDS:
   Before sending, check reminder_log:
   If entry exists for this job_id + 
   reminder_type: skip
   
6. RESPECT QUIET HOURS:
   Never send SMS between 9 PM and 8 AM
   Email can send any time
